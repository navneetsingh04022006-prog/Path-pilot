# Setup Guide

> Local development setup for Path Pilot.  
> Follow this guide during **Phase 0** completion.

---

## Prerequisites

| Tool | Version | Purpose |
|------|---------|---------|
| Node.js | 18+ | Frontend tooling |
| npm or pnpm | Latest | Package management |
| Python | 3.11+ | Backend API (FastAPI) |
| Git | Latest | Version control |
| Supabase account | Free tier | Database + Auth |

Optional (Phase 5+):

- Google Cloud account (Gemini API)
- GitHub personal access token

---

## 1. Clone the Repository

```bash
git clone <repository-url>
cd path-pilot
```

---

## 2. Environment Configuration

Copy the environment template:

```bash
cp .env.example .env
```

Fill in required values:

| Variable | Required | Description |
|----------|----------|-------------|
| `DATABASE_URL` | Yes (Phase 0+) | PostgreSQL connection string |
| `SUPABASE_URL` | Yes (Phase 0+) | Supabase project URL |
| `SUPABASE_ANON_KEY` | Yes (Phase 0+) | Supabase anonymous key |
| `GEMINI_API_KEY` | Yes (Phase 4+) | Google Gemini API key |
| `SECRET_KEY` | Yes (Phase 0+) | Backend secret for signing |
| `JWT_SECRET` | Yes (Phase 2+) | JWT token secret |

> Never commit `.env` to version control.

---

## 3. Supabase Setup (Phase 0)

1. Create a project at [supabase.com](https://supabase.com).
2. Copy the project URL and anon key to `.env`.
3. Enable Email authentication in Supabase Auth settings.
4. Run database migrations (when available in `backend/migrations/`).

---

## 4. Frontend Setup (Phase 0)

> **Status:** Not yet initialized. Run these steps when Phase 0 frontend setup begins.

```bash
cd frontend
npm install
```

Expected dependencies (per [TECH_STACK.md](./TECH_STACK.md)):

- react, react-dom
- vite
- tailwindcss
- react-router-dom
- framer-motion
- recharts
- lucide-react

Start development server:

```bash
npm run dev
```

Frontend runs at: `http://localhost:5173`

---

## 5. Backend Setup (Phase 0)

> **Status:** Not yet initialized. Run these steps when Phase 0 backend setup begins.

```bash
cd backend
python -m venv venv

# Windows
venv\Scripts\activate

# macOS/Linux
source venv/bin/activate

pip install -r requirements.txt
```

Start development server:

```bash
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

Backend runs at: `http://localhost:8000`

API docs (auto-generated by FastAPI): `http://localhost:8000/docs`

---

## 6. Verify Installation

After Phase 0 completion, verify:

- [ ] Frontend loads at `http://localhost:5173`
- [ ] Backend health check responds at `http://localhost:8000/health`
- [ ] Database connection succeeds
- [ ] No console errors in browser or terminal

---

## 8. Common Issues

### Port already in use

Change ports in `.env`:

```
BACKEND_PORT=8001
VITE_API_URL=http://localhost:8001
```

### Supabase connection fails

- Verify `SUPABASE_URL` and keys are correct.
- Check Supabase project is not paused (free tier).

### Python virtual environment

Always activate the venv before running backend commands.
